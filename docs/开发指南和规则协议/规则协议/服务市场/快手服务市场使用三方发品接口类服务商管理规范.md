### 一、目的

1.1【目的及依据】为维护快手服务市场涉及使用三方发品接口应用的发品规范，保障发品到订购方快手店铺、快手平台的合法权益，制定本规范。

1.2【适用范围】本规范适用于快手服务市场中所有涉及使用三方发品接口（服务商可通过下图查询方式在开放平台应用详情中查询自家应用是否涉及｜应用权限为merchant\_item、状态为已获取）的类目及应用、在使用发品接口后的发品数据上报要求及发品质量不合格的后果。

![](https://p2-ec.ecukwai.com/kos/nlav10684/gravity-open-editor/gravity-open-editor-1751007587837.png)

1.3【效力级别】服务商应当遵守《[服务市场管理规范](https://open.kwaixiaodian.com/zone/new/docs/dev?pageSign=32e227de68ea2c81728366cabc6cc8aa1614264216659)》等服务市场规则及本规范的相关规定，本规范有特别规定的事项，适用本规范的规定。

2、定义

使用三方发品接口的应用：是指服务商通过分销或搬家等其他形式发布或者更新商品到用户的快手小店的应用。

### 二、商品来源数据上报考核

#### -上报说明

所有在使用发品接口的应用（包含商品管理、ERP及其他所有三方应用），需要对进行搬家发品、系统发品、分销发品的数据上报，所有使用发品接口的服务商均需使用该接口进行供应信息上报，上报说明如下：

|  |  |
| --- | --- |
| 所有发品的应用，都需使用[商品供应信息上报接口](https://open.kwaixiaodian.com/zone/new/docs/api?name=open.supply.item.sync&version=1)向平台上传发布商品的背后信息，使用时先通过scene标识商品背后的供应来源，枚举值及其背后对应的含义如下：  1-分销铺货：开放SAAS应用，帮助商家上架其他货主的货品到自己店铺进行售卖，如：发布1688、产业带等渠道货源到快手  2-商品搬家：商家将自己在其他平台店铺的商品搬运到快手  3-其他发品类型：其他类型的发品场景，如：商家在三方系统管理自己全渠道的商品，然后将商品发布到快手店铺 | |
| scene枚举项与所需上报的最少必要信息 | |
| 1-分销铺货 | 必填：  1、originItemUrl：搬家来源商品的公开可访问的链接  2、itemId：搬家商品的原始ID  3、kwaiItemId：发布到快手的商品ID  5、platform：平台枚举：1、1688货源，货源来自于1688平台。2、自有货源平台 ，ISV提供铺货工具的同时，也充当供应链公司的角色，自行组织相关货源在工具提供给商家选货。3、货源来自于其他非1688的货源平台&渠道，选择该枚举项时候，需要在“platformName“提供具体的供货信息  6、platformName：货品来源的平台渠道名称：1688、xxx自有货源  7、title：货品的标题信息，原始货品的标题信息  8、priceMax：价格最大值，如商品有多个SKU，取价格最大的SKU的价格，如只有一个SKU，填唯一SKU价格即可  9、priceMin：价格最小值，如商品有多个SKU，取价格最小的SKU的价格，如只有一个SKU，填唯一SKU价格即可  10、imageUrl：货品主图列表第一张图，仅支持https的上传格式  11、categoryId：类目id，货品铺设到快手后，对应铺设对应的快手类目ID  12、seller-name：供货货主的企业、店铺名称  13、seller-platformName：如果platform=3时，需要传入供货的来源渠道具体情况，如“xxx供货平台“、”xxx产业带货源“ |
| 2-商品搬家 | 必填：  1、originItemUrl：搬家来源商品的公开可访问的链接  2、itemId：搬家商品的原始ID  3、kwaiItemId：发布到快手的商品ID  4、platform：商家搬家场景下，该枚举值请默认填入3  5、platformName：商品来源的平台渠道：淘宝、拼多多、小红书 |
| 3-其他发品类型 | 必填：  1、itemId：原始商品ID  2、kwaiItemId：发布到快手的商品ID  3、UploadMethod[新增]：商品的发布方式 |

#### -上报数据考核说明

|  |  |  |
| --- | --- | --- |
| 考核说明 | | |
| * 正式考核日期：自本规则正式生效之日起 * 考核周期：每自然周考核 | | |
| 考核项 | | 考核说明 |
| 考核项1:上报数据完整性 | | 每一次发品&产生分销订单时候，都需要使用接口上传对应的数据，不能漏报，晚报：   * 对于发品来源标识在考核周期内，上报率要>95% * 数据必须在发品后24小时内完成数据上报 |
| 考核项2:上报数据有效 | | 在分销铺货场景上报的数据，必须符合平台要求，要求必填的字段，必须按着平台要求格式正确填写 |
| 考核项3：上报数据真实 | | 所有上报的数据需要基于平台要求真实可靠，对于发品标识，需要根据商家实际发品场景进行正确标识。对于分销铺货业务下上报的货品信息及采购单的货品信息需真实可靠。平台将会不定期进行抽检，对以下情况做严肃处理   * 服务商标识发品场景与实际向商家提供的功能不符（如是铺货场景，但是标识为搬家场景） * 分销场景下，标识的货源信息与实际商家真实铺货货品不符 * 如果上报的渠道是资料包上传，若发品场景选择的是“通过资料包上传”，但后续平台抽检后发现商家是搭配其他爬虫工具（如拼多多、淘宝店铺商品采集工具）进行商品信息获取后然后通过资料包上传的方式发品，也算命中“商品上传信息不真实”，也将视为考核不符合要求 |
| -考核项1、考核项2、若不满足平台考核要求， 发现第一次勒令服务商改正且在服务市场隐藏对应服务工具一个月，并扣除2500元保证金余额，补齐保证金额度后解除隐藏 发现第二次视影响范围做接口限流&强制下架对应服务工具一个月，并扣除5000元保证金余额，补齐保证金额度后才可重新上架服务工具 发现第三次做清退处罚，冻结货款余额并扣除服务商账户剩余全部保证金作为违约金  -考核项3行为发现一次，做直接清退并冻结货款余额和扣除服务商账户剩余全部保证金作为违约金 | | |

### 2、商品发品质量数据考核

所有三方发品应用，需要对自身商品发布质量负责，后续平台将启动对平台核心关注的发品质量目标进行考核，平台将会对服务商应用以下发品质量进行核心的周期性考核

#### -考核说明

|  |  |  |
| --- | --- | --- |
| 考核说明 | | |
|  | | |
| 考核项 | | 考核说明 |
| 考核项1:信息发布功能完整性 | | 使用发品接口的应用，必须在商品发布时，支持：商品多主图发布能力(>3张）、商品3:4图片上传能力、商品视频发布能力 |
| 考核项2:信息发布内容质量合规 | | 使用发品接口的应用，所上传的图片&视频需符合平台基本的底线要求，对于商品图片，需要保障上传的商品图片没有白边及上传至少3张图片 |
| 考核项3:信息发布内容质量提优 | | 平台鼓励服务商为商家提供商品内容辅助工具，帮助商家完成基础质量优化，若发品应用满足以下条件，平台将给予奖励：   * 商品视频填报率：考核周期内，应用视频填报率在所有应用前10% * 商品3:4图片填报率：考核周期内，商品3:4图片填报率在所有应用前10% * 商品短标题填报率：考核周期内，商品短标题填报率在所有应用前10% |
| -考核项1、考核项2：若不满足平台考核要求， 发现第一次勒令服务商改正且在服务市场隐藏对应服务工具一个月，并扣除2500元保证金余额，补齐保证金额度后解除隐藏 发现第二次视影响范围做接口限流&强制下架对应服务工具一个月，并扣除5000元保证金余额，补齐保证金额度后才可重新上架服务工具 第三次做清退处罚，冻结货款余额并扣除服务商账户剩余全部保证金作为违约金  -考核项3：若商家在任意一项达到平台拔优要求，平台将在服务详情给应用赋予“视频发布能力”、“图片处理能力”、“短标题生成能力”的标签，并获得服务市场首页、工作台推荐流量（具体奖励细则后续公布) | | |

### 三、服务商被动清退和主动退出的相关规则

1 服务商违反《[服务市场管理规范](https://open.kwaixiaodian.com/zone/new/docs/dev?pageSign=32e227de68ea2c81728366cabc6cc8aa1614264216659)》或本规则及服务市场其他规则，被平台清退的，按照《[服务市场管理规范](https://open.kwaixiaodian.com/zone/new/docs/dev?pageSign=32e227de68ea2c81728366cabc6cc8aa1614264216659)》有关被动清退的规定处理。

2 服务商被平台清退的，服务商应妥善处理与商家之间的合作，不得以任何方式侵害商家、平台或消费者的合法权益。服务商侵害商家、平台或消费者权益的，平台有权扣收服务商全部保证金，由此给商家、平台或消费者造成损失的，服务商还应赔偿全部损失。

3 服务商申请退出服务市场的，除需符合《[服务市场管理规范](https://open.kwaixiaodian.com/zone/new/docs/dev?pageSign=32e227de68ea2c81728366cabc6cc8aa1614264216659)》及《[快手服务商退出规范](https://open.kwaixiaodian.com/zone/new/faq/detail?cateId=43&pageSign=8dd1cbf1834d4df0f6fc57d06d68c71b1676985362003)》等规定的主动退出条件外，还需满足以下条件：提交退出申请前1个月内无任何维权、投诉等纠纷。

4 服务商申请退出服务市场后6个月内将无法再次入驻。

### 四、附则

本规则于2025年7月17日首次发布，于2025年7月25日生效
